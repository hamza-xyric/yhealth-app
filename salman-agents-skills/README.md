## Usage Guidelines

### Invoking an Agent
```
When starting a new project or task, invoke agents in sequence:

1. Start with Agent 01 (Requirements) to understand the project
2. Use Agent 02 (System Analysis) to validate feasibility
3. Use Agent 03 (Tech Stack) to finalize technologies
4. Continue through the workflow as needed
```

### Agent Handoff Protocol
Each agent produces deliverables that feed into the next:
- Requirements Document â†’ System Analysis
- Feasibility Report â†’ Tech Stack Selection
- Tech Stack â†’ Architecture Design
- Architecture â†’ Database + UI Design
- Designs â†’ Backend + Frontend Development
- Code â†’ Testing & Security
- Tested Code â†’ Deployment

### Best Practices
1. **Sequential Flow:** Follow the workflow order for new projects
2. **Parallel Execution:** Agents 05-06 and 07-08 can work in parallel
3. **Iterative Feedback:** Return to earlier agents when requirements change
4. **Documentation:** Each agent produces exportable documentation
5. **Quality Gates:** Agent 09 validates before Agent 10 deploys
